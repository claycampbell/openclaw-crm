1. Create `apps/web/src/db/schema/saved-views.ts`:
   ```typescript
   import { pgTable, text, timestamp, jsonb, boolean, index } from "drizzle-orm/pg-core";
   import { workspaces } from "./workspace";
   import { users } from "./auth";

   export const savedViews = pgTable("saved_views", {
     id: text("id").primaryKey().$defaultFn(() => crypto.randomUUID()),
     workspaceId: text("workspace_id").notNull()
       .references(() => workspaces.id, { onDelete: "cascade" }),
     objectSlug: text("object_slug").notNull(), // Which object this view applies to (e.g., "deals", "people")
     userId: text("user_id").notNull()
       .references(() => users.id, { onDelete: "cascade" }),
     name: text("name").notNull(),
     filters: jsonb("filters").default({}),  // FilterGroup JSON
     sorts: jsonb("sorts").default([]),       // SortConfig[] JSON
     columns: jsonb("columns").default([]),   // string[] of visible attribute slugs (empty = show all)
     isShared: boolean("is_shared").notNull().default(false),
     createdAt: timestamp("created_at").notNull().defaultNow(),
     updatedAt: timestamp("updated_at").notNull().defaultNow(),
   }, (table) => [
     index("saved_views_workspace_object").on(table.workspaceId, table.objectSlug),
     index("saved_views_user").on(table.userId),
   ]);
   ```

2. Export from schema/index.ts: `export * from "./saved-views";`

3. Run `pnpm db:push`.

4. Create `apps/web/src/services/saved-views.ts`:
   - `listViews(workspaceId: string, objectSlug: string, userId: string)`:
     SELECT views WHERE (workspaceId AND objectSlug) AND (userId = current OR isShared = true).
     This returns the user's own views plus all shared views for the object.
     Order by: user's views first, then shared views, then by name.
   - `createView(params: { workspaceId, objectSlug, userId, name, filters, sorts, columns, isShared })`:
     INSERT and return. Validate name is non-empty (max 50 chars).
   - `updateView(workspaceId: string, viewId: string, userId: string, updates: { name?, filters?, sorts?, columns?, isShared? })`:
     UPDATE only if userId matches (only owner can edit). Return updated view.
   - `deleteView(workspaceId: string, viewId: string, userId: string)`:
     DELETE only if userId matches. Return deleted ID.
   - CRITICAL per Pitfall 14: Filter criteria in shared views are visible to all workspace members who apply them. The isShared flag is set explicitly by the owner. Default is private. Do NOT expose filter JSON for views the user doesn't own in the list endpoint response -- actually, shared views need their filters to work, so they must be included. The privacy model is: private views are invisible to others, shared views are fully visible including their filter criteria.

5. Create `apps/web/src/app/api/v1/objects/[slug]/views/route.ts`:
   - GET: List views for this object (user's own + shared)
   - POST: Create a view for this object. Body: { name, filters, sorts, columns, isShared }

6. Create `apps/web/src/app/api/v1/objects/[slug]/views/[viewId]/route.ts`:
   - PATCH: Update a view (name, filters, sorts, columns, isShared)
   - DELETE: Delete a view (owner only)
