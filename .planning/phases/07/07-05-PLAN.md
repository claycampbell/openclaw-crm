   - Props: `recordId: string`, `recordEmail?: string` (auto-populated To for compose)

   - Fetch threads from `GET /api/v1/records/${recordId}/emails` on mount
   - Layout: Compose button at top, thread list below

2. `email-thread-list.tsx` — Thread list with grouping:
   - Props: `threads: EmailThread[]`, `onSelectThread: (thread: EmailThread) => void`, `selectedThreadId?: string`
   - Render each thread as a card/row showing: subject, last message date (relative time), participant avatars/names, message count badge, latest snippet
   - Unread indicator if any message has `isRead: false`
   - Sort by lastMessageAt descending (most recent first)
   - Use shadcn Card or custom styled divs matching the existing CRM aesthetic
   - Empty state: "No emails synced for this record. Connect your email in Settings > Integrations."

3. `email-thread-view.tsx` — Expanded thread view with lazy body loading:
   - Props: `thread: EmailThread`, `recordId: string`, `onClose: () => void`, `onReply: (threadId: string) => void`
   - Display messages in chronological order (oldest first within thread)
   - Each message shows: from name/email, to/cc pills, date, direction badge (inbound/outbound)
   - Message body: initially shows snippet only. Has an "Expand" button that fetches `GET /api/v1/records/${recordId}/emails/${messageId}/body`
   - Once loaded, render HTML body in a sandboxed container:
     - Use an iframe with `srcdoc` attribute for safe HTML rendering (prevents CSS leaks)
     - Set iframe sandbox="allow-same-origin" (no scripts, no popups)
     - Auto-resize iframe height to content height via a resize observer or postMessage
   - Loading state: skeleton shimmer while body is being fetched
   - Error state: "Unable to load email body. Your email account may need reconnection."
   - Reply button at bottom that triggers onReply with the threadId

Styling: Use Tailwind classes consistent with the existing CRM design. The components should feel like a native part of the record detail page, not a separate email client. Use shadcn/ui primitives (Button, Badge, Card, Separator) where appropriate.
