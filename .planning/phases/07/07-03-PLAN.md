  <action>
Create `apps/web/src/services/record-emails.ts` with two exported functions:

1. `getEmailThreadsForRecord(workspaceId: string, recordId: string, options?: { limit?: number; cursor?: string })`:
   - Query `email_messages` where `recordId = recordId` and `workspaceId = workspaceId`
   - Order by `receivedAt DESC`
   - Group results by `threadId` (in application code, not SQL GROUP BY, since we need per-message data)
   - Return structure:
     ```typescript
     interface EmailThread {
       threadId: string | null;
       subject: string | null;
       lastMessageAt: Date;
       messageCount: number;
       participants: string[]; // unique fromEmail values
       messages: EmailMessageSummary[];
     }
     interface EmailMessageSummary {
       id: string;
       externalId: string;
       provider: "gmail" | "outlook";
       fromEmail: string;
       fromName: string | null;
       toEmails: string[];
       ccEmails: string[];
       subject: string | null;
       snippet: string | null;
       direction: string;
       receivedAt: string; // ISO
       isRead: boolean;
       openedAt: string | null;
       clickedAt: string | null;
     }
     ```
   - Messages with null threadId should each be their own "thread" (standalone messages)
   - Apply cursor-based pagination on threads by `lastMessageAt` + `threadId` composite cursor
   - Default limit: 20 threads

2. `getEmailBody(workspaceId: string, userId: string, messageId: string)`:
   - Look up the `email_messages` row by `id = messageId` and `workspaceId = workspaceId`
   - Determine provider from the row
   - Get a valid token for the user + provider via `getValidToken()`
   - For Gmail: call `gmail.users.messages.get({ userId: "me", id: externalId, format: "full" })`, extract the HTML body from `payload.parts` (look for `text/html` part, decode from base64url), fall back to `text/plain`
   - For Outlook: call `GET /me/messages/{externalId}?$select=body` with the Graph API, extract `body.content` (HTML)
   - Return `{ html: string; plainText?: string }`
   - If token is not available (not connected), throw an error with message "Email provider not connected"
   - Cache bodies briefly: since bodies don't change, consider adding a `bodyHtml` column to `email_messages` that gets populated on first fetch (optional optimization, but helpful). If adding the column, update the schema and check if body is already cached before calling the provider API.

Important: Do NOT store full email bodies by default (schema design decision — snippet only). The lazy-load is intentional to avoid storing potentially large HTML content for thousands of emails. Only cache if explicitly fetched.

1. Create `apps/web/src/app/api/v1/records/[recordId]/emails/route.ts`:
   - GET handler following the standard API pattern:
     ```typescript
     export async function GET(req: NextRequest, { params }: { params: Promise<{ recordId: string }> }) {
       const ctx = await getAuthContext(req);
       if (!ctx) return unauthorized();
       const { recordId } = await params;
       const url = new URL(req.url);
       const limit = parseInt(url.searchParams.get("limit") || "20");
       const cursor = url.searchParams.get("cursor") || undefined;
       const threads = await getEmailThreadsForRecord(ctx.workspaceId, recordId, { limit, cursor });
       return success(threads);
     }
     ```
   - Validate that the record exists and belongs to the workspace before querying emails

2. Create `apps/web/src/app/api/v1/records/[recordId]/emails/[messageId]/body/route.ts`:
   - GET handler:
     ```typescript
     export async function GET(req: NextRequest, { params }: { params: Promise<{ recordId: string; messageId: string }> }) {
       const ctx = await getAuthContext(req);
       if (!ctx) return unauthorized();
       const { messageId } = await params;
       const body = await getEmailBody(ctx.workspaceId, ctx.userId, messageId);
       return success(body);
     }
     ```
   - If body fetch fails due to missing token, return 400 with "Email provider not connected. Please reconnect in Settings > Integrations."
   - If body fetch fails due to provider API error, return 502 with the error message

Note on Next.js 15 params: The params object is a Promise in Next.js 15 App Router dynamic routes. Always `await params` before destructuring.
