Install papaparse: `cd apps/web && pnpm add papaparse && pnpm add -D @types/papaparse`

Create `apps/web/src/services/import.ts`:

1. **fuzzyMatchHeaders(csvHeaders: string[], attributes: {slug: string, title: string}[]): Record<number, string>**
   - For each CSV header, compute similarity against each attribute title and slug
   - Use a simple Levenshtein-based similarity or Dice coefficient (implement inline, ~20 lines -- do NOT add a dependency for this)
   - Match if similarity > 0.6 (configurable threshold)
   - Exact matches (case-insensitive) always win
   - Return mapping of csvColumnIndex -> attributeSlug
   - Handle common aliases: "Name" -> "personal_name", "Email Address" -> "email", "Phone Number" -> "phone"

2. **buildDuplicateLookup(workspaceId: string, objectId: string, matchAttributeSlugs: string[]): Promise<Map<string, string>>**
   - Per Pitfall 10 guidance: pre-load a lookup table BEFORE processing rows
   - Query record_values joined with attributes for the given object, filtering to the specified match attributes (typically email, phone)
   - Build a Map of `normalizedValue -> recordId` for each match attribute
   - Normalize: lowercase, trim whitespace
   - This is ONE query loading all existing values, not per-row queries
   - Return the lookup map

3. **importRecordsWithDedup(params)** where params:
   ```typescript
   {
     workspaceId: string;
     objectId: string;
     rows: Record<string, unknown>[];
     dedupStrategy: "skip" | "update" | "create_all";
     dedupAttributeSlug?: string; // which attribute to check for duplicates (e.g., "email")
     userId: string;
   }
   ```
   - If strategy is "create_all": just call createRecord for each row (existing behavior)
   - If strategy is "skip" or "update":
     a. Call buildDuplicateLookup with the dedup attribute
     b. For each row, check if the dedup attribute value exists in the lookup map
     c. "skip": skip rows where value exists
     d. "update": call updateRecordValues for existing recordId, createRecord for new
   - Return `{ created: number, updated: number, skipped: number, errors: {row: number, message: string}[] }`

Update `apps/web/src/app/api/v1/objects/[slug]/records/import/route.ts`:
- Accept new body fields: `dedupStrategy` (default "create_all"), `dedupAttributeSlug` (optional)
- Call `importRecordsWithDedup` instead of the current per-row createRecord loop
- Return the enhanced result with created/updated/skipped counts
- Keep the 1000 row limit for synchronous imports (Plan 04 handles async for larger)
