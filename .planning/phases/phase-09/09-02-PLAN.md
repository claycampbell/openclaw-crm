2. Create `apps/web/src/app/api/v1/workspace/members/route.ts`:
   - GET handler that returns workspace members for mention autocomplete
   - Call `getAuthContext(req)` for auth
   - Query the `member` table (from Better Auth schema) joined with `user` table to get id, name, email, image
   - Filter by workspaceId from auth context
   - Accept optional `?q=` query param for name/email search filtering (case-insensitive ILIKE)
   - Return `success(members)` with shape: `{ id: string, name: string, email: string, image?: string }[]`
   - CRITICAL: Only return members of the current workspace (per Pitfall 12 from PITFALLS.md -- never query global users table directly)

3. Create `apps/web/src/lib/mentions.ts`:
   - `extractMentionedUserIds(content: unknown): string[]` -- traverses TipTap JSON content tree, finds nodes with type "mention", extracts `attrs.id` values. Returns deduplicated array of user IDs.
   - TipTap mention nodes have shape: `{ type: "mention", attrs: { id: string, label: string } }`
   - Recursively walk `content` array in the JSON structure
   - Export this as a pure utility function (no DB access)

4. Create `apps/web/src/services/mentions.ts`:
   - `notifyMentionedUsers(params: { workspaceId: string, mentionedUserIds: string[], actorId: string, actorName: string, recordId: string, recordName: string, objectSlug: string, context: "note" | "comment" }): Promise<void>`
   - For each userId in mentionedUserIds (excluding actorId -- don't notify yourself):
     - Call `createNotification()` from notifications service with:
       - type: "mention"
       - title: `${actorName} mentioned you in a ${context}`
       - body: `On record: ${recordName}`
       - url: `/objects/${objectSlug}/${recordId}`
       - metadata: `{ actorId, recordId, context }`
   - This function is shared between notes and comments (Plan 03 will reuse it)
