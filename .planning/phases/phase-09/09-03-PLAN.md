1. Create `apps/web/src/db/schema/comments.ts`:
   ```typescript
   import { pgTable, text, timestamp, jsonb, index } from "drizzle-orm/pg-core";
   import { workspaces } from "./workspace";
   import { records } from "./records";
   import { users } from "./auth";

   export const comments = pgTable("comments", {
     id: text("id").primaryKey().$defaultFn(() => crypto.randomUUID()),
     workspaceId: text("workspace_id").notNull()
       .references(() => workspaces.id, { onDelete: "cascade" }),
     recordId: text("record_id").notNull()
       .references(() => records.id, { onDelete: "cascade" }),
     userId: text("user_id").notNull()
       .references(() => users.id, { onDelete: "cascade" }),
     content: text("content").notNull(), // Plain text with @mention markers
     parentId: text("parent_id"), // Self-reference for threading (null = top-level)
     mentions: jsonb("mentions").default([]), // Array of { userId, name } for mentioned users
     createdAt: timestamp("created_at").notNull().defaultNow(),
     updatedAt: timestamp("updated_at").notNull().defaultNow(),
   }, (table) => [
     index("comments_record_idx").on(table.recordId),
     index("comments_workspace_idx").on(table.workspaceId),
     index("comments_parent_idx").on(table.parentId),
   ]);
   ```
   Note: parentId is a soft self-reference (not a foreign key to avoid circular reference issues with Drizzle). Validation is done in the service layer.

2. Export from `apps/web/src/db/schema/index.ts` by adding `export * from "./comments";`

3. Run `pnpm db:push` to create the table.

4. Create `apps/web/src/services/comments.ts`:
   - `listComments(workspaceId: string, recordId: string)`: SELECT all comments for record, ordered by createdAt asc. Join with users table to get commenter name and image. Return flat list (client builds tree from parentId).
   - `createComment(params: { workspaceId, recordId, userId, content, parentId?, mentions? })`: INSERT comment. After insert, if mentions array is non-empty, call `notifyMentionedUsers` from `@/services/mentions` with context "comment". Also look up the actor name from the user table for the notification.
   - `deleteComment(workspaceId: string, commentId: string, userId: string)`: DELETE only if userId matches (only author can delete). Also delete child comments (cascade).
   - All queries scoped by workspaceId.

5. Create `apps/web/src/app/api/v1/records/[id]/comments/route.ts`:
   - GET: `getAuthContext(req)`, then `listComments(ctx.workspaceId, id)`, return `success(comments)`
   - POST: `getAuthContext(req)`, parse body `{ content, parentId?, mentions? }`, validate content is non-empty string, call `createComment(...)`, return `success(comment, 201)`
   - Both scoped to workspace via auth context
