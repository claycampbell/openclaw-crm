  <action>
Update the `ai_generate` handler in `instrumentation.ts` to wrap the generator dispatch with budget and dedup checks. The handler should:

1. Extract payload fields (workspaceId, recordId, documentType, competitorName, contextTier)
2. Check dedup: call `isDuplicate(workspaceId, recordId, assetType, competitorName)`. If true, log and return early.
3. Check budget: call `checkBudget(workspaceId)`. If `allowed === false`, log the reason and return early.
4. Dispatch to the appropriate generator (existing switch from Plan 01)
5. After successful generation, call `recordUsage(workspaceId, contextTier)` to increment the daily counter.

Add dynamic imports at the top of the register() function body:
```typescript
const { checkBudget, recordUsage } = await import("@/services/ai-budget");
const { isDuplicate } = await import("@/services/documents/dedup-guard");
const { ASSET_TIER_MAP } = await import("@/services/documents/asset-registry");
```

The updated handler structure:
```typescript
registerJobHandler("ai_generate", async (payload) => {
  const { workspaceId, recordId, documentType, contextTier, competitorName, ...rest } = payload as AiGeneratePayload & { workspaceId: string };
  if (!workspaceId || !recordId) throw new Error("ai_generate missing workspaceId or recordId");

  const assetType = documentType as AssetType;
  const tier = contextTier ?? ASSET_TIER_MAP[assetType] ?? "light";

  // Guard 1: Deduplication
  const dup = await isDuplicate(workspaceId, recordId, assetType, competitorName);
  if (dup) {
    console.log(`[ai_generate] Skipping duplicate: ${documentType} for record ${recordId} (within 15-min window)`);
    return;
  }

  // Guard 2: Budget
  const budget = await checkBudget(workspaceId);
  if (!budget.allowed) {
    console.log(`[ai_generate] Budget exceeded for workspace ${workspaceId}: ${budget.reason}`);
    return;
  }

  // Dispatch to generator (existing switch)
  switch (documentType) {
    // ... existing cases from Plan 01
  }

  // Record usage after successful generation
  await recordUsage(workspaceId, tier);
});
```

The key principle: guards run BEFORE the expensive LLM call, usage recording happens AFTER. If the generator throws, usage is NOT recorded (the failed attempt does not count against budget since no LLM cost was incurred for the failed draft).
