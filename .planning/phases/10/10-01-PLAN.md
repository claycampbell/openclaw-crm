Create `apps/web/src/db/schema/webhooks.ts` with two tables:

1. `webhook_subscriptions` table:
   - `id` text PK with crypto.randomUUID()
   - `workspace_id` text NOT NULL, FK to workspaces.id with onDelete cascade
   - `url` text NOT NULL (the target endpoint URL)
   - `secret` text (optional HMAC-SHA256 signing secret, stored as-is -- not hashed, needed for signing)
   - `events` text array NOT NULL (allowed values: "record.created", "record.updated", "deal.stage_changed", "deal.closed", "email.received")
   - `enabled` boolean NOT NULL default true
   - `consecutive_failures` integer NOT NULL default 0 (for circuit breaker)
   - `disabled_reason` text (null unless circuit-breaker disabled)
   - `created_by` text, FK to users.id with onDelete set null
   - `created_at` timestamp NOT NULL defaultNow()
   - `updated_at` timestamp NOT NULL defaultNow()

   Indexes: workspace_id, (workspace_id + enabled) for event dispatch lookups.

2. `webhook_deliveries` table:
   - `id` text PK with crypto.randomUUID()
   - `subscription_id` text NOT NULL, FK to webhook_subscriptions.id with onDelete cascade
   - `event_type` text NOT NULL
   - `payload` jsonb NOT NULL
   - `status` text NOT NULL default "pending" (pending/success/failed/dead_letter)
   - `response_code` integer (HTTP status from target)
   - `response_body` text (first 1KB of response for debugging)
   - `attempts` integer NOT NULL default 0
   - `max_attempts` integer NOT NULL default 4 (1 initial + 3 retries)
   - `next_retry_at` timestamp
   - `error_message` text
   - `created_at` timestamp NOT NULL defaultNow()
   - `completed_at` timestamp

   Indexes: subscription_id, (status + next_retry_at) for retry polling, created_at for log browsing.

Export both tables from `apps/web/src/db/schema/index.ts`.

Create `apps/web/src/services/webhooks.ts` with workspace-scoped CRUD:
- `createWebhookSubscription(workspaceId, { url, events, secret?, createdBy })` -- validate URL format, validate events against allowed list, return subscription
- `listWebhookSubscriptions(workspaceId)` -- return all subscriptions for workspace, ordered by createdAt desc
- `getWebhookSubscription(workspaceId, id)` -- single subscription lookup
- `updateWebhookSubscription(workspaceId, id, { url?, events?, secret?, enabled? })` -- partial update, reset consecutive_failures to 0 if re-enabling
- `deleteWebhookSubscription(workspaceId, id)` -- delete by ID within workspace scope
- `testWebhookSubscription(workspaceId, id)` -- send a test ping payload to the URL, return success/failure

All queries MUST include workspace_id in WHERE clause for multi-tenancy.
